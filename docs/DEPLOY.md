# Deployment Guide

Этот документ описывает базовые шаги по развертыванию Teloxide Moderation Runtime.

## Подготовка окружения

1. **Бинарный файл:** Скомпилируйте проект: `cargo build --release`.
2. **Конфигурация:**
   - Скопируйте `config.example.toml` в `config.toml`.
   - Укажите `bot_token` (обязательно).
   - Убедитесь, что пути в `[paths]` доступны для записи ботом.
3. **Структура папок:**
   Создайте необходимые директории:
   ```bash
   mkdir -p data units scripts templates logs
   ```

## Запуск

### Локально
```bash
export TMO_CONFIG=config.toml
./target/release/teloxide-moderation
```

### Docker
Используйте `compose.yaml` в корне проекта:
```bash
docker compose up -d
```

## Шаблоны сообщений

Бот поддерживает систему шаблонов для всех автоматических уведомлений. 

### Система приоритетов:
1. `templates/`: Пользовательские шаблоны (рекомендуется для правок). Эта папка находится в `.gitignore`.
2. `bundled_templates/`: Стандартные шаблоны (поставляются с ботом).

Если вы хотите изменить текст, скопируйте файл из `bundled_templates/` в `templates/` и внесите правки. Бот автоматически будет использовать вашу версию.

### Доступные переменные:
- `{user_link}`, `{admin_link}`: Скрытая ссылка на профиль.
- `{reason}`, `{duration}`: Данные о действии.
- `{warn_count}`, `{warn_limit}`: Статистика предупреждений.
- `{date}`, `{time}`: Время события.
